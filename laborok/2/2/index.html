    2. feladat – Consumer és JSON üzenetek¶

    A valós streaming rendszerekben az üzenetek nem egyszerű szövegek, hanem strukturált adatok – leggyakrabban JSON formátumban. Ebben a feladatban definiálunk egy OrderEvent adatobjektumot, JSON-ként szerializálva küldjük Kafka-ba, majd egy consumerrel olvassuk vissza és DataFrame-be rendezzük.

    Szerializáció és deszerializáció

    A Kafka bináris protokollt használ: minden üzenet bytes típusú. Az alkalmazás feladata eldönteni, hogyan alakítja az adatot byte-okká küldéskor (szerializáció), és vissza küldés után (deszerializáció). A JSON azért népszerű, mert szöveg alapú, olvasható, és szinte minden rendszer ismeri – cserébe valamivel terjengősebb, mint a bináris formátumok (Avro, Protobuf).

    JSON producer¶

    1. Definiáljuk az OrderEvent adatosztályt, és küldjünk 20 JSON üzenetet:

      from dataclasses import dataclass, asdict  # Adatosztály dekorátor és dict-té alakítás
from kafka import KafkaProducer
import json, random, time
from datetime import datetime

BOOTSTRAP = "kafka:9092"

# @dataclass automatikusan generál __init__, __repr__ és __eq__ metódusokat
# A típusannotációk (int, float, str) csak dokumentációs célúak Pythonban – nem kényszerítik ki a típust
@dataclass
class OrderEvent:
    order_id: int        # Rendelés egyedi azonosítója
    customer_id: int     # Ügyfél azonosítója (1–100 között szimulált)
    product_id: int      # Termék azonosítója (1–50 között szimulált)
    quantity: int        # Rendelt mennyiség
    unit_price: float    # Egységár forintban
    event_time: str      # Az esemény időbélyege ISO 8601 formátumban (pl. "2024-03-14T10:30:00")

def make_order(i: int) -> OrderEvent:
    """Véletlenszerű OrderEvent-et generál az i-edik rendeléshez."""
    return OrderEvent(
        order_id=i,
        customer_id=random.randint(1, 100),    # Véletlen ügyfél 1–100 közül
        product_id=random.randint(1, 50),       # Véletlen termék 1–50 közül
        quantity=random.randint(1, 5),           # 1–5 darab
        unit_price=round(random.uniform(500, 50000), 2),  # 500–50000 Ft, 2 tizedesre kerekítve
        event_time=datetime.utcnow().isoformat(),  # Aktuális UTC idő, ISO 8601 stringként
    )

# value_serializer: minden egyes elküldött értéket ez a függvény alakít bytes-szá
# lambda v: json.dumps(v) → JSON string (pl. '{"order_id": 1, ...}')
# .encode("utf-8")        → bytes (pl. b'{"order_id": 1, ...}')
producer = KafkaProducer(
    bootstrap_servers=BOOTSTRAP,
    value_serializer=lambda v: json.dumps(v).encode("utf-8"),
)

for i in range(1, 21):
    event = make_order(i)
    # asdict(event): az @dataclass objektumot Python dict-té alakítja
    # A value_serializer automatikusan JSON-ná + bytes-szá konvertálja
    producer.send("orders", value=asdict(event))
    time.sleep(0.05)  # 50 ms szünet üzenetek között – elkerüli a burst küldést

producer.flush()
producer.close()
print("20 OrderEvent elküldve JSON formátumban.")

    Consumer és DataFrame¶

    1. Írjunk egy consumert, amely beolvassa az összes eddig elküldött üzenetet, és pandas DataFrame-be rendezi:

      from kafka import KafkaConsumer
import pandas as pd

def safe_deserialize(v):
    """Bytes → Python objektum; ha nem JSON, visszaadja a nyers stringet."""
    if not v:
        return None
    try:
        return json.loads(v.decode("utf-8"))
    except (json.JSONDecodeError, UnicodeDecodeError):
        return v.decode("utf-8", errors="replace")

# KafkaConsumer első pozícionális argumenntumai a feliratkozni kívánt topicok nevei
consumer = KafkaConsumer(
    "orders",                        # Iratkozz fel az "orders" topicra
    bootstrap_servers=BOOTSTRAP,
    auto_offset_reset="earliest",    # Ha nincs korábbi offset, a legelejéről kezdje az olvasást
                                    # (alternatíva: "latest" – csak az újonnan érkező üzeneteket olvassa)
    enable_auto_commit=False,        # Ne mentse automatikusan az offsetet – mi irányítunk
    # value_deserializer: bytes → Python objektum
    value_deserializer=safe_deserialize,
    consumer_timeout_ms=3000,        # Ha 3 másodpercig nem jön új üzenet, lépjen ki a for ciklusból
)

records = []
for msg in consumer:
    # msg.value: a deszerializált üzenet (itt Python dict)
    # Az 1. feladatból string üzenetek is vannak a topicban – azokat kihagyjuk
    if isinstance(msg.value, dict):
        row = msg.value.copy()        # Másolat, hogy ne módosítsuk az eredetit
        row["partition"] = msg.partition  # Melyik partícióból jött (0, 1 vagy 2)
        row["offset"] = msg.offset        # Hanyadik üzenet volt abban a partícióban
        records.append(row)

consumer.close()  # Kapcsolat bezárása – felszabadítja a partíció-hozzárendelést

# pd.DataFrame(records): lista of dict-ből táblázat – minden dict egy sor, a kulcsok az oszlopnevek
df = pd.DataFrame(records)
print(f"Beolvasott JSON üzenetek száma: {len(df)}")
df.head(5)  # Az első 5 sor megjelenítése táblázatban

    Partíció-elosztás vizualizálása¶

    1. Nézzük meg, hogyan oszlottak el az üzenetek a partíciók között:

      # groupby("partition"): partíció értéke szerint csoportosít
# .size(): minden csoportban megszámolja az elemeket
# .reset_index(name="üzenet_szám"): a sorozatot DataFrame-mé alakítja, az oszlop neve "üzenet_szám"
part_dist = df.groupby("partition").size().reset_index(name="üzenet_szám")
print("Partíció-elosztás:")
print(part_dist.to_string(index=False))  # index=False: ne jelenjen meg a sor indexe

      Partíció-hozzárendelés

      Partíció-kulcs nélküli küldésnél a Kafka sticky partitioner szerint osztja el az üzeneteket: rövid időszakonként választ egy partíciót, és addig arra küld. Ez hatékonyabb a round-robin-nél, de kevésbé egyenletes az elosztás. Ha minden üzenetnek egyformán kell eloszlania, adjunk meg key= paramétert a producer.send() hívásnál.
