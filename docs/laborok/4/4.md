# 4. feladat – Great Expectations

A Great Expectations (GE) egy nyílt forráskódú adatminőség-keretrendszer, amellyel deklaratív módon fogalmazzuk meg az adatokkal szemben támasztott elvárásainkat, és automatikusan érvényesítjük őket. A pipeline-ba integrálva automatikus DQ (Data Quality) kapuként működik: ha az adat nem felel meg az elvárásoknak, a pipeline megáll és riasztást küld.

!!! note "GE alapfogalmak (1.x API)"
    * **Data Context** – a GE projekt gyökere. `mode='file'` esetén a `gx/` mappában tárol mindent.
    * **Data Source** – az adatforrás (pandas, Spark, SQL). A laboron pandas.
    * **Data Asset** – az adatforrásból kinyert dataset logikai neve.
    * **Batch Definition** – meghatározza, hogyan kérjük le az adatot (egész DataFrame, partíció, stb.).
    * **Expectation Suite** – elvárások gyűjteménye. Verziókontrollálható, dokumentált.
    * **Expectation** – egyetlen elvárás: pl. `ExpectColumnValuesToNotBeNull(column='order_id')`.
    * **Validation Definition** – összekapcsolja a suite-ot a batch definition-nel.
    * **Checkpoint** – futtatható validációs egység: elvárásokat alkalmaz egy batch-re és menti az eredményt.
    * **Data Docs** – automatikusan generált HTML dokumentáció az elvárásokról és eredményekről.

!!! warning "GE 1.x és 0.x API különbség"
    A labor a GE **1.x API**-t használja. A GE 0.x-ből ismert `context.sources`, `context.get_validator()`, `context.add_or_update_checkpoint()` metódusok **nem léteznek** az 1.x-ben. Az 1.x API: `context.data_sources`, `context.suites`, `context.validation_definitions`, `context.checkpoints`.

42. A negyedik cellában definiáljuk az elvárásokat. A GE 1.x-ben az elvárásokat közvetlenül a suite objektumhoz adjuk – nem validator-on keresztül:

    ```python
    # Expectation Suite létrehozása
    suite = context.suites.add(gx.ExpectationSuite(name=SUITE_NAME))

    # ── Tábla szintű elvárások ────────────────────────────────────────────────

    # A táblában legalább 1, legfeljebb 10 millió sor legyen
    suite.add_expectation(
        gx.expectations.ExpectTableRowCountToBeBetween(min_value=1, max_value=10_000_000)
    )
    # Az oszlopok pontosan ezek legyenek, ebben a sorrendben
    suite.add_expectation(
        gx.expectations.ExpectTableColumnsToMatchOrderedList(
            column_list=['order_id', 'customer_id', 'order_date', 'status', 'total_amount']
        )
    )

    # ── Completeness elvárások ────────────────────────────────────────────────

    # order_id és customer_id soha nem lehet NULL
    suite.add_expectation(gx.expectations.ExpectColumnValuesToNotBeNull(column='order_id'))
    suite.add_expectation(gx.expectations.ExpectColumnValuesToNotBeNull(column='customer_id'))
    suite.add_expectation(gx.expectations.ExpectColumnValuesToNotBeNull(column='total_amount'))

    # ── Uniqueness elvárás ────────────────────────────────────────────────────

    # order_id értékek egyediek – elsődleges kulcs tulajdonság
    suite.add_expectation(gx.expectations.ExpectColumnValuesToBeUnique(column='order_id'))

    # ── Validity elvárások ────────────────────────────────────────────────────

    # total_amount mindig pozitív (negatív összeg adathiba)
    suite.add_expectation(
        gx.expectations.ExpectColumnValuesToBeBetween(
            column='total_amount', min_value=0.01, max_value=100_000_000
        )
    )
    # status csak ezeket az értékeket veheti fel (az ETL csak 'completed'-et enged át)
    suite.add_expectation(
        gx.expectations.ExpectColumnValuesToBeInSet(
            column='status', value_set=['completed']
        )
    )
    # Az átlagos rendelési összeg reális tartományban legyen
    suite.add_expectation(
        gx.expectations.ExpectColumnMeanToBeBetween(
            column='total_amount', min_value=1000.0, max_value=500_000.0
        )
    )

    print(f"Elvárások definiálva: {len(suite.expectations)} db")
    for exp in suite.expectations:
        exp_type = getattr(exp, 'type', getattr(exp, 'expectation_type', 'unknown'))
        kwargs   = getattr(exp, 'kwargs', {})
        col      = kwargs.get('column', 'TABLE')
        print(f"  [{col:15}] {exp_type}")
    ```

    !!! note "Az elvárások típusai"
        | Típus | Példa elvárás |
        |-------|--------------|
        | **Completeness** | `ExpectColumnValuesToNotBeNull` |
        | **Uniqueness** | `ExpectColumnValuesToBeUnique` |
        | **Validity – range** | `ExpectColumnValuesToBeBetween` |
        | **Validity – set** | `ExpectColumnValuesToBeInSet` |
        | **Validity – regex** | `ExpectColumnValuesToMatchRegex` |
        | **Statistical** | `ExpectColumnMeanToBeBetween`, `ExpectColumnStdevToBeBetween` |
        | **Schema** | `ExpectTableColumnsToMatchOrderedList` |
        | **Volume** | `ExpectTableRowCountToBeBetween` |

    ```python
    # Hibás adat: duplikált order_id, NULL customer_id, negatív összeg, érvénytelen status
    df_bad = pd.DataFrame({
        'order_id':    [1, 2, 2, 4, 5],          # Duplikált: 2
        'customer_id': [101, None, 103, 104, 105], # NULL: 2. sor
        'order_date':  ['2024-01-15', '2024-01-16', '2024-01-17', '2024-01-18', '2024-01-19'],
        'status':      ['completed', 'completed', 'completed', 'completed', 'cancelled'],  # 5. sor: nem completed
        'total_amount': [12500.0, 3200.0, -100.0, 450.0, 15000.0],  # Negatív: 3. sor
    })

    result_bad = checkpoint.run(batch_parameters={'dataframe': df_bad})

    print(f"\n{'='*55}")
    print(f"Checkpoint: {'✅ PASSED' if result_bad.success else '❌ FAILED'}")
    print(f"{'='*55}")

    val_results  = list(result_bad.run_results.values())
    expectations = val_results[0].results if val_results else []
    passed = sum(1 for r in expectations if r.success)
    failed = len(expectations) - passed
    print(f"Elvárások: {passed} teljesült, {failed} sikertelen\n")

    for r in expectations:
        icon = '✅' if r.success else '❌'
        cfg      = r.expectation_config
        exp_type = getattr(cfg, 'type', getattr(cfg, 'expectation_type', 'unknown'))
        col      = (cfg.kwargs or {}).get('column', 'TABLE')
        # Sikertelen esetén írjuk ki a konkrét hibát
        if not r.success:
            result_detail = r.result
            print(f"  {icon} [{col:15}] {exp_type}")
            if 'unexpected_count' in result_detail:
                print(f"       → {result_detail['unexpected_count']} érvénytelen érték"
                      f" ({result_detail.get('unexpected_percent', 0):.1f}%)")
        else:
            print(f"  {icon} [{col:15}] {exp_type}")
    ```

    !!! note "Mit mutat az `unexpected_count`?"
        Minden sikertelen elvárás részletes eredményt tartalmaz:
        * `unexpected_count` – hány értéket sértett meg az elvárást
        * `unexpected_percent` – a sértő értékek aránya
        * `unexpected_values` – a konkrét sértő értékek listája (első 20 db)
        * `partial_unexpected_list` – mintavétel a sértő értékekből

### DQ Gate – pipeline integráció

46. A nyolcadik cellában valósítsuk meg a DQ Gate mintát: a GE checkpoint-ot beépítjük a pipeline logikájába, és hiba esetén leállítjuk a futást:

    ```python
    class DQPipelineGate:
        """
        Data Quality kapu a Bronze→Silver és Silver→Gold átmeneteken.
        Ha a validáció sikertelen, RuntimeError-t dob – az Airflow/Prefect task FAILED lesz.
        """

        def __init__(self, gx_context, checkpoint_name: str, suite_name: str,
                     batch_definition, fail_threshold: float = 1.0):
            """
            fail_threshold: ha a sikerességi arány ez alá esik, hiba keletkezik.
            1.0 = minden elvárásnak teljesülnie kell (szigorú mód).
            0.8 = 80%-os teljesülés elég (megengedő mód).
            """
            self.ctx             = gx_context
            self.checkpoint_name = checkpoint_name
            self.fail_threshold  = fail_threshold
            self.batch_definition = batch_definition
            self.suite_name      = suite_name

        def run(self, df: pd.DataFrame, stage: str = 'silver') -> bool:
            checkpoint = self.ctx.checkpoints.get(self.checkpoint_name)
            result     = checkpoint.run(batch_parameters={'dataframe': df})

            val_results  = list(result.run_results.values())
            expectations = val_results[0].results if val_results else []
            total  = len(expectations)
            passed = sum(1 for r in expectations if r.success)
            ratio  = passed / total if total > 0 else 0.0

            print(f"[DQ Gate / {stage}] {passed}/{total} elvárás teljesült ({ratio*100:.1f}%)")

            if ratio < self.fail_threshold:
                failed_exps = [r for r in expectations if not r.success]
                fail_details = "; ".join(
                    getattr(r.expectation_config, 'type',
                            getattr(r.expectation_config, 'expectation_type', '?'))
                    for r in failed_exps
                )
                raise RuntimeError(
                    f"DQ Gate FAILED ({stage}): {passed}/{total} teljesült. "
                    f"Sikertelen elvárások: {fail_details}"
                )

            print(f"  → DQ Gate PASSED – adat továbbítható a {stage} rétegbe")
            return True


    gate = DQPipelineGate(
        gx_context=context,
        checkpoint_name='orders_checkpoint',
        suite_name=SUITE_NAME,
        batch_definition=batch_definition,
    )

    print("=== Valid adaton ===")
    gate.run(df_valid, stage='silver')

    print("\n=== Hibás adaton ===")
    try:
        gate.run(df_bad, stage='silver')
    except RuntimeError as e:
        print(f"Pipeline megállítva: {e}")
    ```
