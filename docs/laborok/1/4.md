# 4. feladat: SCD Type 2 – Ügyfél dimenzió

A valóságban az ügyfelek adatai változnak: városba költöznek, szegmenst váltanak. Ha csak felülírjuk a régi értékeket (SCD Type 1), elveszítjük azt az információt, hogy az ügyfél melyik szegmensben volt egy korábbi rendelés pillanatában. Az **SCD Type 2** (Slowly Changing Dimension Type 2) minden változáskor egy új sort szúr be, a régi sort lezárja — így a teljes előzmény visszakereshető.

### SCD1 vs SCD2 összehasonlítás

3. Olvassuk el az alábbi magyarázatot (ez egy Markdown cella a notebookban):

    > **SCD Type 1 – Felülírás:** Az új értékkel felülírjuk a meglévő sort. Egyszerű, de az előzmény elvész. Jó: ha az előzmény nem releváns (pl. elírás javítása).
    >
    > **SCD Type 2 – Előzmény megőrzése:** Minden változásnál a régi sort lezárjuk (`valid_to`, `is_current=FALSE`), és egy új sort szúrunk be (`valid_from=most`, `valid_to='9999-12-31'`, `is_current=TRUE`). Jó: ha visszamenőlegesen is tudni kell, hogy az ügyfél melyik szegmensben volt egy korábbi tranzakció pillanatában.

### Inicializálás

4. Hozzuk létre az SCD2 dimenziótáblát a customers staging tábla alapján:

    ```python
    duck.execute(f"""
        CREATE OR REPLACE TABLE stg_customers
        AS SELECT * FROM read_parquet('{RAW}/customers.parquet')
    """)

    duck.execute("""
        CREATE OR REPLACE TABLE dim_customer_scd2 AS
        SELECT
            customer_id,
            name,
            email,
            city,
            segment,
            created_at                       AS valid_from,
            CAST('9999-12-31' AS TIMESTAMP)  AS valid_to,
            TRUE                             AS is_current,
            1                                AS version
        FROM stg_customers
    """)

    total = duck.execute("SELECT COUNT(*) FROM dim_customer_scd2").fetchone()[0]
    current = duck.execute(
        "SELECT COUNT(*) FROM dim_customer_scd2 WHERE is_current = TRUE"
    ).fetchone()[0]

    ```

### SCD2 frissítési logika

5. Implementáljunk egy segédfüggvényt, amely elvégzi az SCD2 frissítést:

    ```python
    def apply_scd2_change(duck, customer_id, new_segment, new_city):
        """
        SCD Type 2 frissítés:
          1. Régi aktuális sor lezárása (valid_to = most, is_current = FALSE)
          2. Új sor beszúrása (valid_from = most, valid_to = 9999, is_current = TRUE, version++)
        """
        now = datetime.now().isoformat()

        # 1. Régi sor lezárása
        duck.execute(f"""
            UPDATE dim_customer_scd2
            SET valid_to   = CAST('{now}' AS TIMESTAMP),
                is_current = FALSE
            WHERE customer_id = {customer_id}
              AND is_current = TRUE
        """)

        # 2. Új sor beszúrása
        duck.execute(f"""
            INSERT INTO dim_customer_scd2
            SELECT
                customer_id,
                name,
                email,
                '{new_city}',
                '{new_segment}',
                CAST('{now}' AS TIMESTAMP),
                CAST('9999-12-31' AS TIMESTAMP),
                TRUE,
                version + 1
            FROM dim_customer_scd2
            WHERE customer_id = {customer_id}
              AND is_current = FALSE
            ORDER BY version DESC
            LIMIT 1
        """)

    print("apply_scd2_change() függvény definiálva.")
    ```

### Ellenőrzés

7. Ellenőrizzük az SCD2 tábla konzisztenciáját, és kérdezzük le a módosított ügyfelek előzményét:

    ```python
    total = duck.execute("SELECT COUNT(*) FROM dim_customer_scd2").fetchone()[0]
    current = duck.execute(
        "SELECT COUNT(*) FROM dim_customer_scd2 WHERE is_current = TRUE"
    ).fetchone()[0]
    print(f"Összes sor: {total} (volt: 500, +5 új verzió)")
    print(f"is_current = TRUE: {current} (változatlanul 500)\n")

    changed_ids = [c[0] for c in changes]
    result = duck.execute(f"""
        SELECT
            customer_id,
            name,
            city,
            segment,
            strftime(valid_from, '%Y-%m-%d %H:%M') AS valid_from,
            CASE WHEN valid_to > CAST('9998-01-01' AS TIMESTAMP)
                 THEN 'aktuális'
                 ELSE strftime(valid_to, '%Y-%m-%d %H:%M')
            END AS valid_to,
            is_current,
            version
        FROM dim_customer_scd2
        WHERE customer_id IN ({','.join(str(i) for i in changed_ids)})
        ORDER BY customer_id, version
    """).df()
    print("=== 5 módosított ügyfél – mindkét verzió ===\n")
    print(result.to_string(index=False))
    ```

    !!! warning "Ellenőrzési pontok"
        Az SCD2 implementáció helyes, ha:

        * Az összes sor száma `500 + 5 = 505`
        * Az `is_current = TRUE` sorok száma pontosan `500` marad
        * Minden módosított ügyfélnél a régi sor `valid_to` értéke kitöltött (nem `9999-12-31`)
        * Minden módosított ügyfélnél az új sor `version = 2`

### Időbeli rekonstrukció

8. Kérdezzük le, hogy az érintett ügyfelek melyik szegmensben voltak a változásuk előtt:

    ```python
    # Keressük meg az érintett ügyfelek rendeléseit és rekonstruáljuk,
    # milyen szegmensben voltak az adott rendelés dátumán
    duck.execute(f"""
        CREATE OR REPLACE TABLE stg_orders
        AS SELECT * FROM read_parquet('{RAW}/orders.parquet')
    """)

    reconstruction = duck.execute(f"""
        SELECT
            o.order_id,
            o.order_date::DATE          AS order_date,
            d.customer_id,
            d.name                      AS customer_name,
            d.segment                   AS segment_at_order_time,
            d.version
        FROM stg_orders o
        JOIN dim_customer_scd2 d
          ON o.customer_id = d.customer_id
         AND o.order_date::TIMESTAMP >= d.valid_from::TIMESTAMP
         AND o.order_date::TIMESTAMP <  d.valid_to::TIMESTAMP
        WHERE o.customer_id IN ({','.join(str(i) for i in changed_ids)})
        ORDER BY d.customer_id, o.order_date DESC
        LIMIT 20
    """).df()

    print("=== Időbeli rekonstrukció – melyik szegmensben volt az ügyfél a rendelés pillanatában? ===\n")
    print(reconstruction.to_string(index=False))
    ```

    !!! note "SCD2 időbeli JOIN"
        A `valid_from <= order_date < valid_to` feltétel biztosítja, hogy minden rendeléshez pontosan azt a dimenziósort választjuk, amely a rendelés pillanatában aktuális volt. Ez a historikus elemzés alapja — anélkül, hogy az aktuális ügyféladatokat felülírnánk.
