# 3. feladat – Dagster

A Dagster az asset-alapú orkestráció élharcosa. A kérdés nem „milyen lépéseket hajtsunk végre?", hanem „milyen adatokat akarunk előállítani és hogyan?".

!!! note "Dagster alapfogalmak"
    * **@asset** – egy adatterméket előállító függvény. A paraméternevek = upstream asset nevek.
    * **Lineage** – automatikusan épül fel a `@asset` dekorátorokból, nincs explicit `>>` operátor.
    * **Materialize** – az asset tartalmának tényleges előállítása.
    * **DailyPartitionsDefinition** – időalapú particionálás: minden naphoz külön materializáció.
    * **IO Manager** – az asset kimenetének mentési/betöltési logikája (pl. Parquet, Delta Lake, S3).
    * **Definitions** – az összes asset, job és resource összefogója.
    * **Freshness policy** – meghatározza, mikor számít az asset „elavultnak" (pl. 24 óránál régebbi).

### Az első Asset

34. A második cellában definiáljuk az első Dagster asset-et:

    ```python
    from dagster import asset, Definitions, materialize, AssetExecutionContext
    import pandas as pd

    @asset(name="szam_lista", description="Páros számok listája", group_name="demo")
    def szam_lista(context: AssetExecutionContext) -> pd.DataFrame:
        szamok = list(range(2, 21, 2))
        df = pd.DataFrame({'szam': szamok, 'negyzet': [x**2 for x in szamok]})
        context.log.info(f"Előállítva: {len(df)} sor")
        return df

    @asset(name="szam_statisztika", description="Statisztikák", group_name="demo")
    def szam_statisztika(szam_lista: pd.DataFrame) -> pd.DataFrame:
        # A paraméter neve ('szam_lista') = upstream asset neve → Dagster automatikus lineage!
        return pd.DataFrame({
            'mutato': ['darab', 'atlag', 'sum', 'max'],
            'ertek':  [len(szam_lista), szam_lista['szam'].mean(),
                       szam_lista['szam'].sum(), szam_lista['szam'].max()]
        })

    defs   = Definitions(assets=[szam_lista, szam_statisztika])
    result = materialize([szam_lista, szam_statisztika])
    print(f"Materializálás: {result.success}")
    print(result.output_for_node("szam_statisztika").to_string(index=False))
    ```

    ```python
    @asset(name="completed_orders", description="Teljesített rendelések (Silver)", group_name="etl")
    def completed_orders(raw_orders: pd.DataFrame) -> pd.DataFrame:
        df = raw_orders[raw_orders['status'] == 'completed'].copy()
        df['order_date']   = pd.to_datetime(df['order_date'])
        df['total_amount'] = df['total_amount'].astype(float)
        return df

    @asset(name="monthly_revenue", description="Havi bevétel (Gold)", group_name="etl")
    def monthly_revenue(completed_orders: pd.DataFrame) -> pd.DataFrame:
        df = completed_orders.copy()
        df['year_month'] = df['order_date'].dt.to_period('M').astype(str)
        result = (
            df.groupby('year_month')
            .agg(order_count=('order_id', 'count'), revenue=('total_amount', 'sum'),
                 customers=('customer_id', 'nunique'))
            .reset_index()
        )
        result['revenue'] = result['revenue'].round(2)
        return result
    ```

    **Miért `nunique()` a customers-nél?** Az `order_id.count()` a rendelések számát adja, míg a `customer_id.nunique()` az egyedi ügyfelek számát – hány különböző vásárló rendelt adott hónapban. Ez a vásárlói aktivitás mérésének alapja.

### Particionált asset – időalapú szeletelés

37. A Dagster `DailyPartitionsDefinition` lehetővé teszi, hogy az asset-eket napokra bontva materializáljuk. Ez az iparági standard nagy adatkészletek inkrementális betöltésénél:

    ```python
    from dagster import (asset, Definitions, materialize,
                         DailyPartitionsDefinition, AssetExecutionContext)
    import pandas as pd
    from sqlalchemy import create_engine
    from datetime import date

    # Napi particionálás: 2024-01-01-től napjainkig
    daily_partitions = DailyPartitionsDefinition(start_date="2024-01-01")

    @asset(
        name="daily_orders",
        description="Napi rendelések – particionált asset",
        group_name="partitioned",
        partitions_def=daily_partitions,   # Ez teszi az assetet particionálttá
    )
    def daily_orders(context: AssetExecutionContext) -> pd.DataFrame:
        """
        A context.partition_key tartalmazza az aktuális partíció dátumát (pl. '2024-03-15').
        Így minden partíció csak a saját napjának adatait kéri le – inkrementális betöltés!
        """
        partition_date = context.partition_key   # pl. '2024-03-15'
        context.log.info(f"Partíció: {partition_date}")

        engine = create_engine(OLTP_CONN)
        df = pd.read_sql(f"""
            SELECT o.id AS order_id, o.customer_id, o.order_date,
                   SUM(oi.quantity * oi.unit_price) AS total_amount
            FROM orders o JOIN order_items oi ON oi.order_id = o.id
            WHERE o.order_date = '{partition_date}'   -- Csak az adott nap adatai!
            GROUP BY o.id, o.customer_id, o.order_date
        """, engine)

        context.log.info(f"Nap {partition_date}: {len(df)} rendelés")
        return df

    part_defs = Definitions(assets=[daily_orders])

    # Egyetlen partíció materializálása – a valóságban a Dagster scheduler végzi
    result = materialize(
        [daily_orders],
        partition_key="2024-06-15",   # Konkrét nap materializálása
    )
    print(f"Partíció materializálva: {result.success}")
    df_day = result.output_for_node("daily_orders")
    print(f"  2024-06-15: {len(df_day)} rendelés")
    ```

    !!! note "Miért particionálás?"
        Particionálás nélkül minden futáskor az összes historikus adatot újra be kell tölteni – ez lassan növekvő költséget jelent. Particionált assetnél csak az érintett nap fut le: ha egy nap hibásan töltődött be, csak azt az egy partíciót kell újra materializálni, nem az egész historikát. Ez az **inkrementális betöltés** alapja.

    !!! tip "Dagster UI – partíció nézet"
        A `dagster dev` elindítása után az asset oldalán megjelenik a naptár nézet: melyik napok materializáltak (zöld), melyeknek hiányzik az adata (szürke), melyeknél volt hiba (piros). Ez az egyik leghasznosabb Dagster UI funkció éles rendszerekben.
