    8. feladat – End-to-end streaming pipeline¶

    Az előző feladatokban az egyes komponenseket – producert, validátort, windowing aggregátort, DLQ kezelőt, Parquet exportálót – külön-külön implementáltuk. Ebben a zárófeladatban ezeket egyetlen StreamingPipeline osztályba kell összefogni. A pipeline önállóan fut, és 200 üzenet feldolgozása után leáll.

    3. Pipeline architektúra – az osztály felépítése:

      StreamingPipeline
├── _produce_events(n)         → N db OrderEvent → "orders-pipeline" topic
├── _consume_and_validate()    → JSON Schema validáció, érvénytelen → DLQ
├── _aggregate_windows()       → TumblingWindowAggregator (10 s ablakok)
├── _export_parquet()          → aggregált ablakok → Parquet fájl
└── start(n)                   → sorban: produce → consume → aggregate → export

      Osztálymetódusok konvenciója

      Az aláhúzással kezdődő metódusok (_produce_events, _consume_and_validate, stb.) Python konvenció szerint „privát" metódusok – ezeket csak az osztályon belülről szabad hívni (a start() metódusból). Az osztály felhasználója csak a start() metódust hívja.

      Megoldási útmutató a _produce_events metódushoz 
      def _produce_events(self, n: int) -> None:
    producer = KafkaProducer(
        bootstrap_servers=BOOTSTRAP,
        value_serializer=lambda v: json.dumps(v).encode("utf-8"),
    )
    for i in range(n):
        order_id = 500 + i
        msg = {
            "order_id":    order_id,
            "customer_id": random.randint(1, 100),
            "product_id":  random.randint(1, 50),
            "quantity":    random.randint(1, 5),
            "unit_price":  round(random.uniform(500, 50000), 2),
            "event_time":  datetime.now(timezone.utc).isoformat(),
        }
        if i % 5 == 0:          # Minden 5. üzenet szándékosan hibás
            msg["quantity"] = -1
        producer.send(PIPELINE_TOPIC, value=msg)
        time.sleep(0.05)
    producer.flush()
    producer.close()
    print(f"{n} esemény elküldve (köztük {n // 5} szándékosan hibás).")
      Megoldási útmutató a _consume_and_validate metódushoz 
      def _consume_and_validate(self) -> list:
    consumer = KafkaConsumer(
        PIPELINE_TOPIC,
        bootstrap_servers=BOOTSTRAP,
        group_id="pipeline-validator",
        auto_offset_reset="earliest",
        enable_auto_commit=True,
        value_deserializer=lambda v: json.loads(v.decode("utf-8")),
        consumer_timeout_ms=8000,
    )
    dlq_producer = KafkaProducer(
        bootstrap_servers=BOOTSTRAP,
        value_serializer=lambda v: json.dumps(v).encode("utf-8"),
    )
    valid_msgs = []
    valid = invalid = 0
    for msg in consumer:
        if isinstance(msg.value, dict):
            try:
                jsonschema.validate(instance=msg.value, schema=ORDER_SCHEMA)
                valid_msgs.append(msg.value)
                valid += 1
            except jsonschema.ValidationError as e:
                dlq_producer.send(DLQ_TOPIC, value={
                    "original": msg.value,
                    "error": e.message,
                    "ts": datetime.utcnow().isoformat(),
                })
                invalid += 1
    dlq_producer.flush()
    dlq_producer.close()
    consumer.close()
    print(f"Validálva: {valid} OK, {invalid} DLQ")
    return valid_msgs
