# 2. feladat – SCD Type 2 (Slowly Changing Dimensions)

A valós adattárházakban a dimenziók nem maradnak örökre ugyanolyanok – ügyfelek várost váltanak, termékek kategóriát kapnak, részlegek átszerveződnek. Az SCD (Slowly Changing Dimension, lassan változó dimenzió) módszerek különbözőképpen kezelik ezeket a változásokat. Az **SCD Type 2** megőrzi a változások teljes történetét: minden változáskor új sort szúr be érvényességi intervallummal, és a régi sort lezárja.

!!! note "Az SCD típusok összehasonlítása"
    * **SCD Type 1** – Felülírás: egyszerűen frissíti a régi értéket. Nem marad meg a korábbi állapot. Gyors, de nincs historikus adatsor.
    * **SCD Type 2** – Új sor verzióval: minden változáskor új sor keletkezik, a régi sor lezárásra kerül (`is_current=False`). A teljes történet megmarad – ez a legelterjedtebb módszer.
    * **SCD Type 3** – Előző érték oszlopban: `city_current` és `city_previous` oszlopok. Csak az egy lépéssel korábbi állapot marad meg.

    Három mező adja az SCD Type 2 logikáját:
    * `valid_from` – mettől érvényes ez a verzió (pl. `2024-01-15`)
    * `valid_to` – meddig érvényes (aktív rekordnál: `9999-12-31` – „mindig érvényes" konvenció)
    * `is_current` – az aktuális verzió jelzője (`True` / `False`)

        ```python
        for row in updates:
            cid = row['customer_id']

            # Keressük az aktuális (is_current=TRUE) verziót
            existing = con.execute("""
                SELECT customer_key, name, city, country
                FROM dim_customer_scd2
                WHERE customer_id = ? AND is_current = TRUE
            """, [cid]).fetchone()

            if existing is None:
                # Új ügyfél – egyszerű INSERT
                max_key += 1
                con.execute("""
                    INSERT INTO dim_customer_scd2
                        (customer_key, customer_id, name, city, country,
                         valid_from, valid_to, is_current)
                    VALUES (?, ?, ?, ?, ?, ?, DATE '9999-12-31', TRUE)
                """, [max_key, cid, row['name'], row['city'], row['country'], effective_date])
                print(f"  INSERT: customer_id={cid} ({row['name']}) – új ügyfél")
            else:
                old_key, old_name, old_city, old_country = existing
                changed = (
                    old_name    != row['name']    or
                    old_city    != row['city']    or
                    old_country != row['country']
                )
                if changed:
                    # 1. Régi sor lezárása: valid_to = effective_date - 1 nap
                    con.execute("""
                        UPDATE dim_customer_scd2
                        SET valid_to = ?, is_current = FALSE
                        WHERE customer_key = ?
                    """, [prev_day, old_key])

                    # 2. Új sor az új értékekkel
                    max_key += 1
                    con.execute("""
                        INSERT INTO dim_customer_scd2
                            (customer_key, customer_id, name, city, country,
                             valid_from, valid_to, is_current)
                        VALUES (?, ?, ?, ?, ?, ?, DATE '9999-12-31', TRUE)
                    """, [max_key, cid, row['name'], row['city'], row['country'], effective_date])
                    print(f"  UPDATE: customer_id={cid} – {old_city} → {row['city']}")
                else:
                    print(f"  SKIP:   customer_id={cid} – nincs változás")
        ```

        **Miért `effective_date - 1 nap` a valid_to?** Ha a változás hatályba lép `2024-04-01`-én, akkor a régi verzió utolsó érvényes napja `2024-03-31`. A `BETWEEN valid_from AND valid_to` feltétellel ez pontosan érhető el: `2024-03-31 BETWEEN 2024-01-01 AND 2024-03-31` → TRUE, `2024-04-01 BETWEEN 2024-01-01 AND 2024-03-31` → FALSE.
