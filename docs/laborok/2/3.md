## 3. feladat – Sémavalidáció

Valós streaming rendszerekben az üzenet-séma inkonzisztenciái az egyik leggyakoribb hibaforrás. Ha a producer nem küld kötelező mezőt, vagy hibás típusú értéket ad meg, a downstream fogyasztók félremehetnek. Ebben a feladatban JSON Schema-val validált producert írunk: csak érvényes üzenet kerülhet a topicba.

!!! note "JSON Schema"
    A JSON Schema egy nyílt szabvány ([json-schema.org](https://json-schema.org/)), amellyel JSON dokumentumok struktúráját és tartalmát írhatjuk le deklaratívan. Megadhatjuk, hogy egy objektumnak milyen kötelező mezői vannak, milyen típusúak, mi a minimális/maximális értékük, stb. A Python `jsonschema` csomag ezek alapján tud validálni egyetlen függvényhívással.

    ```python
    !pip install -q kafka-python jsonschema pandas

    import json, time, random
    from datetime import datetime
    from dataclasses import dataclass, asdict
    from kafka import KafkaProducer, KafkaConsumer
    from kafka.admin import KafkaAdminClient, NewTopic
    import jsonschema          # JSON dokumentumok sémával való validálásához
    import pandas as pd

    BOOTSTRAP = "kafka:9092"

    # Az ORDER_SCHEMA egy Python dict, amely a JSON Schema specifikációt írja le.
    # Ez alapján tudja a jsonschema csomag eldönteni, hogy egy üzenet érvényes-e.
    ORDER_SCHEMA = {
        "type": "object",           # Az üzenet egy JSON objektum (kulcs-érték párok)
        # required: ezek a mezők kötelezőek – ha bármelyik hiányzik, az üzenet érvénytelen
        "required": ["order_id", "customer_id", "product_id", "quantity", "unit_price", "event_time"],
        "properties": {             # Az egyes mezők részletes szabályai
            "order_id":    {"type": "integer", "minimum": 1},        # Egész, legalább 1
            "customer_id": {"type": "integer", "minimum": 1},
            "product_id":  {"type": "integer", "minimum": 1},
            "quantity":    {"type": "integer", "minimum": 1, "maximum": 100},  # 1–100 közötti egész
            "unit_price":  {"type": "number",  "minimum": 0.01},     # Pozitív szám (float is lehet)
            "event_time":  {"type": "string"},                        # Bármilyen string elfogadott
        },
        "additionalProperties": False,  # Nem engedünk meg ismeretlen mezőket az üzenetben
    }

    print("JSON Schema definiálva.")
    print(f"Kötelező mezők: {ORDER_SCHEMA['required']}")
    ```

    ??? tip "Megoldási útmutató – kattints a tippekért"
        A `jsonschema.validate()` különleges: ha az üzenet megfelel a sémának, **nem ad vissza semmit** (None), ha nem felel meg, akkor `jsonschema.ValidationError` kivételt dob. Ez azt jelenti, hogy a sikeres eset a „semmi sem történt" eset.

        A függvény felépítése:
        ```python
        def validate_message(msg: dict) -> bool:
            try:
                jsonschema.validate(instance=msg, schema=ORDER_SCHEMA)
                # Ha idáig eljutottunk, a validáció sikeres volt
                return True
            except jsonschema.ValidationError:
                # A kivétel jelzi, hogy az üzenet nem felel meg a sémának
                return False
        ```

        **Miért try/except és nem if/else?** A `jsonschema.validate()` API-t így tervezték: kivételdobással jelzi a hibát, nem visszatérési értékkel. Ez a Python-ban bevett minta – EAFP (Easier to Ask Forgiveness than Permission).

    ??? tip "Megoldási útmutató – kattints a tippekért"
        A `send_validated` függvény felépítése:
        ```python
        def send_validated(msg: dict) -> None:
            if validate_message(msg):
                # Érvényes üzenet: elküldjük a validált topicra
                producer.send("orders-validated", value=msg)
                print(f"[OK] order_id={msg.get('order_id')}")

                # Érvénytelen üzenet: eldobjuk, de naplózzuk
                print(f"[INVALID] {msg}")
        ```

        **Figyelem:** a `producer` változó a függvényen kívül van definiálva – Python closure-ként elérhető belülről. Ha `UnboundLocalError`-t kapsz, ellenőrizd, hogy a `producer = KafkaProducer(...)` sor a függvénydefiníció **előtt** fut-e.
