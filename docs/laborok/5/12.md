# Hibaelhárítás

**`BucketAlreadyExists` hiba S3 létrehozáskor**  
→ A bucket név globálisan egyedi, valaki már használja. Ellenőrizd a NEPTUN változót, legyen tényleg a te kódod.

**Glue job `FAILED` állapot**  
→ AWS Console → Glue → Jobs → Runs → Error message → CloudWatch Logs. Leggyakoribb ok: rossz S3 path, LabRole jogosultság.

**Athena `TABLE_NOT_FOUND`**  
→ A Glue Crawler (F3) még nem futott le, vagy a database neve eltér. Ellenőrizd: `DB_NAME = f"smart_city_{NEPTUN.lower()}"`.

**`ParamValidationError: Invalid type for parameter ..., value: None`**  
→ Kitöltetlen `None` placeholder maradt a kódban. Keresd a `# TODO:` kommenteket és írd be a valódi értéket.

**Athena `mismatched input 'é'` / `mismatched input 'ő'` stb.**  
→ Az Athena SQL parser (Presto) nem fogad el ékezetes karaktereket idézőjel nélküli azonosítókban (alias, oszlopnév). Csak ASCII karaktereket használj SQL aliasokban: `avg_sebesseg_kmh` ✓, `avg_sebesség_kmh` ✗.

**Athena `COLUMN_NOT_FOUND: Column '...' cannot be resolved` (HAVING / ORDER BY)**  
→ Presto/Athena-ban a `SELECT`-ben definiált alias nem hivatkozható `HAVING`-ben, és feltételes aggregátumoknál (`CASE WHEN`) `ORDER BY`-ban sem. Megoldás: CTE-vel (`WITH agg AS (...)`) emeld ki az aggregációt, majd a külső lekérdezésben szűrj és rendezz az alias-ra — ott már valódi oszlopként látszik.

**Athena `HIVE_INVALID_METADATA: duplicate columns`**  
→ A Glue Crawler duplikált oszlopokat talált: a CSV-ben lévő `year`/`month` mező ütközik a Hive partícióból (`year=2025/month=01/`) automatikusan létrehozott azonos nevű partícióoszlopokkal. Megoldás:
1. Töröld a raw bucket tartalmát: AWS Console → S3 → `[prefix]-smart-city-raw` → `sensors/` mappa törlése
2. Futtasd újra az F2 cellát (feltölt a javított script alapján, `year`/`month` nélkül)
3. Töröld és hozd létre újra a Crawlert (F3), vagy a Glue Console-on töröld a régi táblát és futtasd újra

**Lambda nem indul el**  
→ `lmb.get_function_configuration()["State"]` nem `Active`. Várj 30 másodpercet, majd próbáld újra.

**Terraform `Error: Invalid provider configuration`**  
→ A credentials lejárt. Másold újra a Learner Lab credentials-t `~/.aws/credentials`-be, majd `docker compose run --rm terraform plan` újra.

**`docker compose run --rm terraform destroy` nem töröl mindent**  
→ S3 bucket-eket csak akkor tud törölni a Terraform, ha üresek. Töröld manuálisan a tartalmát az AWS Console-on, majd futtasd újra a destroy-t.
