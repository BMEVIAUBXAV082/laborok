# 1. feladat – Kafka környezet és első producer

Az Apache Kafka egy elosztott üzenetközvetítő rendszer, amelyet magas áteresztőképességű, valós idejű adatfolyamok kezelésére terveztek. Eredetileg a LinkedIn fejlesztette ki belső loggyűjtésre, ma már az egyik legelterjedtebb streaming platform. Ebben a feladatban megismerjük a Kafka alapfogalmait, létrehozunk egy topicot, és elküldjük az első üzeneteinket.

!!! note "Kafka alapfogalmak"
    * **Topic** – logikai csatorna, amelyre az üzenetek kerülnek; a neve azonosítja (pl. `orders`). Olyan, mint egy adatbázistábla neve, de sorba rendezett, és az elemek nem törlődnek azonnal.
    * **Partition** – egy topic több, párhuzamosan olvasható részre osztható; minden üzenet pontosan egy partícióba kerül. Több partíció → több párhuzamos consumer → nagyobb áteresztőképesség.
    * **Offset** – egy partíción belül az üzenet sorszáma (0, 1, 2, …). A consumer ezt tárolja, hogy legközelebb onnan folytathassa, ahol abbahagyta.
    * **Producer** – az üzeneteket küldő fél; nem tudja, melyik consumer olvassa majd.
    * **Consumer** – az üzeneteket fogadó fél; megmondja, melyik topic melyik partícióját olvassa, és hol tartott utoljára.
    * **Bootstrap server** – az a Kafka broker cím, amelyhez a client először csatlakozik; innen megtudja a cluster többi tagjának elérhetőségét.

### Topic létrehozása

12. A második cellában importáljuk a szükséges modulokat, és hozzuk létre az `orders` topicot 3 partícióval:

    ```python
    from kafka.admin import KafkaAdminClient, NewTopic  # Admin műveletek: topic létrehozás/törlés
    from kafka import KafkaProducer                     # Üzenetküldéshez
    import json, time                                   # JSON szerializáció és várakozás

    BOOTSTRAP = "kafka:9092"  # A Kafka broker Docker hálózaton belüli neve és portja

    # KafkaAdminClient: admin szintű műveletek elvégzésére (topic kezelés, ACL-ek stb.)
    admin = KafkaAdminClient(bootstrap_servers=BOOTSTRAP)

    # Lekérjük a már létező topicok listáját – ha 'orders' már van, töröljük
    # (Ez biztosítja, hogy újrafuttatáskor tiszta állapotból induljunk)
    existing = admin.list_topics()
    if "orders" in existing:
        admin.delete_topics(["orders"])  # Töröljük a régi topicot az összes üzenetével együtt
        time.sleep(2)                    # Várunk, amíg a törlés véglegesedik a brokeren

    # NewTopic: egy topic leírója – itt adjuk meg a nevet, partíciók számát és replikációt
    # replication_factor=1: minden partíciónak csak 1 másolata van (mivel csak 1 brokerünk van)
    admin.create_topics([
        NewTopic(name="orders", num_partitions=3, replication_factor=1)
    ])
    print("Topic 'orders' létrehozva, 3 partícióval.")
    admin.close()  # Az admin kapcsolatot mindig zárjuk le, hogy ne foglaljon erőforrást
    ```

    !!! note "Miért 3 partíció?"
        A partíciószám határozza meg, hogy hány consumer olvashat párhuzamosan ugyanabból a topicból egy consumer group-on belül. Ha 3 partíciónk van, akkor legfeljebb 3 consumer dolgozhat egyszerre. A 4. feladatban ezt a párhuzamosságot ki is fogjuk próbálni.

### Első üzenetek küldése

13. Hozzunk létre egy egyszerű producert, és küldjünk el 10 üzenetet:

    ```python
    # KafkaProducer: az üzenetküldő objektum
    # bootstrap_servers: a broker elérési útja – innen tanulja meg a cluster topológiáját
    producer = KafkaProducer(bootstrap_servers=BOOTSTRAP)

    for i in range(10):
        # Az üzenetet byte-okká kell alakítani – a Kafka bináris protokollt használ
        msg = f"Rendelés #{i+1}".encode("utf-8")  # Python string → bytes (UTF-8 kódolással)

        # send() aszinkron: azonnal visszatér egy Future objektummal, nem vár a brokerre
        future = producer.send("orders", value=msg)

        # future.get(timeout=10): szinkron várakozás a visszaigazolásra (max 10 másodpercig)
        # metadata tartalmazza: melyik partícióba került, mi lett az offset-je
        metadata = future.get(timeout=10)
        print(f"  Üzenet #{i+1} → partition={metadata.partition}, offset={metadata.offset}")

    # flush(): megvárja, amíg az összes küldés folyamatban lévő üzenet megérkezik a brokerhez
    # Enélkül előfordulhat, hogy az utolsó üzenetek elvesznek, ha a program idő előtt leáll
    producer.flush()
    producer.close()  # Kapcsolat bezárása, erőforrások felszabadítása
    print("\nMind a 10 üzenet sikeresen elküldve.")
    ```
