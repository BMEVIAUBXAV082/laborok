# Feladat 8 – Terraform init + S3 + Glue infrastruktúra

Az előző 7 feladatban boto3-szal programozottan hoztuk létre az infrastruktúrát. Most ugyanezt a pipeline-t Terraform kóddal írjuk le — ez az **Infrastructure as Code** (IaC) megközelítés.

**A Terraform Docker-ben fut** — a `docker-compose.yml` `terraform` service-én keresztül, külön terminálból. A Jupyter notebook csak referenciaként lehet nyitva.

### Terraform parancsok futtatása Dockerből

A **Terraform nem igényel lokális telepítést** — a `hashicorp/terraform:1.9` image tartalmazza. Minden Terraform parancsot a `week07/labor/docker/` könyvtárból kell kiadni:

```bash
# Általános forma:
docker compose run --rm terraform <parancs>

# Példák:
docker compose run --rm terraform version
docker compose run --rm terraform init
docker compose run --rm terraform plan
```

!!! tip "Rövidítés (opcionális)"
    Ha sokat gépelnél, definiálj alias-t a terminálban:
    ```bash
    # Linux/macOS
    alias tf='docker compose run --rm terraform'
    # Windows PowerShell
    function tf { docker compose run --rm terraform @args }
    ```
    Ezután `tf init`, `tf plan`, `tf apply` stb.

### 8.1 Terraform könyvtár létrehozása

Terminálban, a projekt repository gyökerében:

```bash
mkdir -p week07/labor/terraform/modules/streaming
```

A `docker-compose.yml` a `../terraform` könyvtárat mountolja `/workspace`-ként, tehát a Terraform fájlokat a `week07/labor/terraform/` mappába kell helyezni.

### 8.2 `providers.tf`

```hcl
terraform {
  required_version = ">= 1.9"
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
  # FIGYELEM: S3 backend TILOS Learner Lab-ban (credentials lejárnak)
  # Lokális state-t használunk
}

provider "aws" {
  region = var.aws_region
  default_tags {
    tags = {
      Project   = var.project_name
      Week      = "06"
      ManagedBy = "terraform"
      Neptun    = var.neptun_code
    }
  }
}
```

!!! tip "Automatikus betöltés"
    A `terraform.tfvars` fájlt a Terraform **automatikusan betölti** — nincs szükség `-var-file` argumentumra. A többi változónak (`shard_count`, `firehose_buffer_seconds`) van alapértelmezett értéke a `variables.tf`-ben, ezért nem kell explicit megadni. Az `environment` értékét a workspace neve határozza meg (ld. `locals` blokk a `main.tf`-ben).

### 8.5 `main.tf` – S3 + Glue

```hcl
locals {
  name_prefix = "${var.project_name}-${var.neptun_code}"
  environment = terraform.workspace  # "default" → dev, "staging" → staging

  bucket_zones = toset([
    "raw",
    "processed",
    "streaming",
    "query-results",
  ])

  lab_role_arn = "arn:aws:iam::${data.aws_caller_identity.current.account_id}:role/LabRole"
}

data "aws_caller_identity" "current" {}

# ── S3 Data Lake (for_each: 4 bucket egy resource blokkal) ──────────────────
resource "aws_s3_bucket" "smart_city" {
  for_each = local.bucket_zones
  bucket   = "${local.name_prefix}-smart-city-${each.key}"
}

resource "aws_s3_bucket_lifecycle_configuration" "raw_glacier" {
  bucket = aws_s3_bucket.smart_city["raw"].id

  rule {
    id     = "raw-sensors-to-glacier"
    status = "Enabled"
    filter { prefix = "sensors/" }
    transition {
      days          = 30
      storage_class = "GLACIER"
    }
  }
}

# ── Glue Data Catalog ────────────────────────────────────────────────────────
resource "aws_glue_catalog_database" "smart_city" {
  name        = "smart_city_${replace(var.neptun_code, "-", "_")}_tf"
  description = "Smart City IoT szenzor adatok (Terraform IaC)"
}

resource "aws_glue_crawler" "sensors" {
  name          = "${local.name_prefix}-sensor-crawler-tf"
  role          = local.lab_role_arn
  database_name = aws_glue_catalog_database.smart_city.name
  description   = "Smart City CSV szenzor adatok felderítése"

  s3_target {
    path = "s3://${aws_s3_bucket.smart_city["raw"].id}/sensors/"
  }

  schema_change_policy {
    update_behavior = "UPDATE_IN_DATABASE"
    delete_behavior = "LOG"
  }
}
```

### 8.6 Terraform futtatása Dockerből

Minden parancsot a `week07/labor/docker/` könyvtárból adj ki:

```bash
cd week07/labor/docker

# Inicializálás (provider letöltés)
docker compose run --rm terraform init

# Terv megtekintése
docker compose run --rm terraform plan

# Apply: S3 bucket-ek + Glue infrastruktúra
docker compose run --rm terraform apply

# Figyeld meg a kimenetet:
# Apply complete! Resources: X added, 0 changed, 0 destroyed.
```

!!! tip "Targeted apply (haladó)"
    A `-target` kapcsolóval elvben csak egyes erőforrásokat provizionálhatsz, de Windows/PowerShell alatt a `docker compose run` a pontot tartalmazó argumentumot (`aws_s3_bucket.smart_city`) rosszul tokenizálja. Ha mégis ki akarod próbálni, add meg a resource-t külön space-szel elválasztva: `docker compose run --rm terraform apply -target aws_s3_bucket.smart_city` — így a `-target` és az érték két külön argumentum, nem `=`-jellel összekapcsolt egy.
