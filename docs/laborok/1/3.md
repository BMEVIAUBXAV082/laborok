# 3. feladat: DuckDB transzformációk – Staging, Cleaned, Aggregated

A **Transform** fázisban az extract-ált nyers adatokat üzleti logika alapján megtisztítjuk és aggregáljuk. Ebben a feladatban a [DuckDB](https://duckdb.org/)-t használjuk: egy in-process analitikai adatbázist, amely közvetlenül tud Parquet fájlokból olvasni, és vektorizált végrehajtása révén rendkívül gyors aggregációs lekérdezéseket képes futtatni.

### Notebook létrehozása

1. Hozzunk létre egy új notebookot `feladat3_transform.ipynb` néven.

2. Az első cellában telepítsük a csomagokat és inicializáljuk a kapcsolatokat:

    ```python

    import pandas as pd

    duck = duckdb.connect()
    RAW = "/home/jovyan/data/extract/full"
    print("DuckDB kapcsolat kész (in-memory).")
    ```

### Staging réteg

3. Töltsük be a Parquet fájlokat DuckDB staging táblákba:

    ```python
    tables = ["customers", "products", "orders", "order_items"]

    print("=== Staging réteg ===\n")
    for table in tables:
        duck.execute(f"""
            CREATE OR REPLACE TABLE stg_{table}
            AS SELECT * FROM read_parquet('{RAW}/{table}.parquet')
        """)
        cnt = duck.execute(f"SELECT COUNT(*) FROM stg_{table}").fetchone()[0]
        print(f"  stg_{table:<15} → {cnt:>6,} sor")

    print("\nstg_orders séma:")
    for col in duck.execute("DESCRIBE stg_orders").fetchall():
        print(f"  {col[0]:<20} {col[1]}")
    ```

    !!! note "Staging réteg célja"
        A staging táblák a nyers adatokat tartalmazzák, változtatás nélkül. Ez a kiindulási alap a downstream transzformációkhoz. A `CREATE OR REPLACE` biztosítja az idempotenciát: a pipeline bármikor újrafuttatható, az eredmény ugyanaz lesz.

### Cleaned réteg

4. Hozzuk létre a `cleaned_orders` táblát JOIN-nal, szűréssel és kategorizálással:

    ```python
    duck.execute("""
        CREATE OR REPLACE TABLE cleaned_orders AS
        SELECT
            o.order_id,
            o.customer_id,
            c.name        AS customer_name,
            c.segment     AS customer_segment,
            c.city        AS customer_city,
            o.order_date,
            o.status,
            o.total_amount,
            CASE
                WHEN o.total_amount >= 100000 THEN 'nagy'
                WHEN o.total_amount >= 20000  THEN 'közepes'
                ELSE 'kis'
            END AS order_size
        FROM stg_orders o
        JOIN stg_customers c ON o.customer_id = c.customer_id
        WHERE o.status != 'cancelled'
    """)
    cnt = duck.execute("SELECT COUNT(*) FROM cleaned_orders").fetchone()[0]
    print(f"cleaned_orders: {cnt:,} sor (cancelled kiszűrve)\n")
    print(duck.execute("SELECT * FROM cleaned_orders LIMIT 3").df())
    ```

### Aggregált rétegek

5. Hozzuk létre a napi értékesítési összesítőt:

    ```python
    duck.execute("""
        CREATE OR REPLACE TABLE daily_sales AS
        SELECT
            CAST(order_date AS DATE)           AS order_day,
            COUNT(*)                           AS order_count,
            SUM(total_amount)                  AS total_revenue,
            AVG(total_amount)::NUMERIC(12, 2)  AS avg_order_value,
            COUNT(DISTINCT customer_id)        AS unique_customers
        FROM cleaned_orders
        GROUP BY CAST(order_date AS DATE)
        ORDER BY order_day
    """)
    cnt = duck.execute("SELECT COUNT(*) FROM daily_sales").fetchone()[0]
    print(f"daily_sales: {cnt} nap\n")
    print(duck.execute(
        "SELECT * FROM daily_sales ORDER BY order_day DESC LIMIT 5"
    ).df())
    ```

6. Hozzuk létre a termékkategória statisztikát:

    ```python
    duck.execute("""
        CREATE OR REPLACE TABLE category_stats AS
        SELECT
            p.category,
            COUNT(DISTINCT oi.order_id)                     AS order_count,
            SUM(oi.quantity)                                AS total_quantity,
            SUM(oi.quantity * oi.unit_price)::NUMERIC(12,2) AS total_revenue,
            AVG(oi.unit_price)::NUMERIC(12,2)               AS avg_price
        FROM stg_order_items oi
        JOIN stg_products p ON oi.product_id = p.product_id
        GROUP BY p.category
        ORDER BY total_revenue DESC
    """)
    print("=== Kategória statisztika ===\n")
    print(duck.execute("SELECT * FROM category_stats").df().to_string(index=False))
    ```

    !!! tip "DuckDB és Parquet"
        A DuckDB a `read_parquet()` függvénnyel közvetlenül fájlokból olvas — nincs szükség külön betöltési lépésre. A vektorizált oszlopos végrehajtás miatt a DuckDB aggregációs lekérdezéseknél tipikusan **5–50×-es** sebességelőnnyel bír a PostgreSQL-lel szemben, különösen nagy adathalmazokon.

### Szegmens × rendelésméret kereszttábla

7. Vizsgáljuk meg, hogy az egyes ügyfélszegmensekben hogyan oszlanak el a rendelésméret kategóriák:

    ```python
    pivot = duck.execute("""
        SELECT
            customer_segment,
            SUM(CASE WHEN order_size = 'kis'     THEN 1 ELSE 0 END) AS kis_rendelesek,
            SUM(CASE WHEN order_size = 'közepes' THEN 1 ELSE 0 END) AS kozepes_rendelesek,
            SUM(CASE WHEN order_size = 'nagy'    THEN 1 ELSE 0 END) AS nagy_rendelesek,
            COUNT(*)                                                  AS osszes
        FROM cleaned_orders
        GROUP BY customer_segment
        ORDER BY customer_segment
    """).df()

    print("=== Szegmens × rendelésméret kereszttábla ===\n")
    print(pivot.to_string(index=False))

    # Százalékos arányok
    pivot_pct = pivot.copy()
    for col in ["kis_rendelesek", "kozepes_rendelesek", "nagy_rendelesek"]:
        pivot_pct[col] = (pivot[col] / pivot["osszes"] * 100).round(1).astype(str) + "%"
    print("\n=== Százalékos eloszlás ===\n")
    print(pivot_pct[["customer_segment", "kis_rendelesek", "kozepes_rendelesek", "nagy_rendelesek"]].to_string(index=False))
    ```

### Havi trendek LAG() window functionnel

8. Elemezzük a havi bevételi trendeket, és számítsuk ki a hónapról-hónapra változást:

    ```python
    monthly = duck.execute("""
        WITH monthly_sales AS (
            SELECT
                DATE_TRUNC('month', order_date)::DATE  AS month,
                COUNT(*)                               AS order_count,
                SUM(total_amount)::NUMERIC(14,2)       AS revenue
            FROM cleaned_orders
            GROUP BY DATE_TRUNC('month', order_date)
        )
        SELECT
            month,
            order_count,
            revenue,
            LAG(revenue)  OVER (ORDER BY month)  AS prev_month_revenue,
            ROUND(
                (revenue - LAG(revenue) OVER (ORDER BY month))
                / LAG(revenue) OVER (ORDER BY month) * 100, 1
            )                                    AS mom_change_pct
        FROM monthly_sales
        ORDER BY month DESC
        LIMIT 12
    """).df()

    print("=== Havi bevételi trendek (utolsó 12 hónap) ===\n")
    print(monthly.to_string(index=False))
    ```

    !!! note "LAG() window function"
        A `LAG(revenue) OVER (ORDER BY month)` az előző sor értékét adja vissza az ablak sorrendjében. Így egyetlen lekérdezéssel kiszámítható az előző havi bevétel és a százalékos változás — nincs szükség self-JOIN-ra.
