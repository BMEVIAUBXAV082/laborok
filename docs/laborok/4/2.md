# 2. feladat – Prefect

A Prefect 2020-ban jelent meg „Python-first" orkestráció igényére válaszul. Ahol az Airflow DAG szintaxist igényel, ott a Prefect `@flow` és `@task` dekorátorokkal bővíti a meglévő Python kódot. A Prefect szerver opcionális.

!!! note "Prefect vs Airflow – főbb különbségek"
    | Szempont | Airflow | Prefect |
    |---------|---------|---------|
    | **Paradigma** | DAG: speciális Python szintaxis | Python-first: dekorátorok |
    | **Szerver** | Kötelező | Opcionális |
    | **Task definíció** | `PythonOperator(python_callable=fn)` | `@task` dekorátor |
    | **Adatátadás** | XCom (DB-n át) | Közvetlen Python return |
    | **Retry** | `default_args retries` | `@task(retries=3)` |
    | **Cache** | Nincs beépített | `cache_key_fn=task_input_hash` |
    | **Subflow** | Nincs | `@flow`-ból hívható `@flow` |
    | **Hibakezelés** | `on_failure_callback` | `on_failure_hooks` |
    | **Tanulási görbe** | Meredek | Alacsony |

### Az első Flow

27. A második cellában valósítsunk meg egy egyszerű Prefect flow-t:

    ```python
    from prefect import flow, task
    from prefect.logging import get_run_logger

    @task
    def lekerdez(url: str) -> str:
        logger = get_run_logger()
        logger.info(f"Task fut, bemenet: {url}")
        return f"Adat érkezett: {url}"

    @task
    def feldolgoz(adat: str) -> dict:
        logger = get_run_logger()
        logger.info("Feldolgozás...")
        # Közvetlenül kapja a bemenetet – nincs XCom, nincs xcom_pull()
        return {"eredmeny": adat.upper(), "hossz": len(adat)}

    @flow(name="elso-flow", log_prints=True)
    def elso_flow(url: str = "http://example.com"):
        adat  = lekerdez(url)
        eredm = feldolgoz(adat)   # Közvetlen Python return – így adj át taskok között!
        print(f"Flow végeredmény: {eredm}")
        return eredm

    result = elso_flow()
    print(f"Visszatérési érték: {result}")
    ```

### Retry és cache

28. A harmadik cellában:

    ```python
    from prefect import flow, task
    from prefect.tasks import task_input_hash
    from datetime import timedelta

    attempt_count = 0

    @task(
        retries=3,
        retry_delay_seconds=2,
        cache_key_fn=task_input_hash,          # Cache kulcs: a bemenet hash-e
        cache_expiration=timedelta(minutes=5), # 5 percig érvényes a cache
    )
    def megbizhatalan_task(bemenet: int) -> int:
        global attempt_count
        attempt_count += 1
        print(f"  Kísérlet #{attempt_count}")
        if attempt_count < 2:
            raise ValueError(f"Szimulált hiba #{attempt_count}")
        return bemenet * 2

    @flow(name="retry-cache-demo", log_prints=True)
    def retry_cache_flow():
        global attempt_count
        attempt_count = 0
        print("=== 1. futás – retry szükséges ===")
        eredm = megbizhatalan_task(21)
        print(f"Eredmény: {eredm}")
        print("\n=== 2. futás – cache-ből ===")
        eredm2 = megbizhatalan_task(21)   # Azonos bemenet → cache találat
        print(f"Eredmény (cache): {eredm2}")

    retry_cache_flow()
    ```

### Subflow – egymásba ágyazott flow-ok

30. A Prefect egyik előnye az Airflow-val szemben, hogy `@flow` egy másik `@flow`-ból hívható. Ez lehetővé teszi a pipeline moduláris felépítését és újrafelhasználhatóságát:

    ```python
    from prefect import flow, task
    import pandas as pd

    @task(log_prints=True)
    def fetch_customers() -> pd.DataFrame:
        """Ügyféladatok lekérdezése (szimulált)."""
        import random
        cities = ['Budapest', 'Debrecen', 'Miskolc']
        return pd.DataFrame({
            'customer_id': range(1, 11),
            'city': [random.choice(cities) for _ in range(10)],
        })

    @task(log_prints=True)
    def fetch_orders(customer_ids: list) -> pd.DataFrame:
        """Rendelések lekérdezése (szimulált)."""
        import random
        return pd.DataFrame({
            'customer_id': [random.choice(customer_ids) for _ in range(30)],
            'amount': [round(random.uniform(1000, 50000), 2) for _ in range(30)],
        })

    # Subflow: önállóan is futtatható, és főflow-ból is hívható
    @flow(name="data-ingestion-subflow", log_prints=True)
    def ingestion_subflow() -> tuple:
        """Adatbefogadási subflow: ügyfelek + rendelések."""
        customers = fetch_customers()
        orders    = fetch_orders(customers['customer_id'].tolist())
        print(f"Subflow kész: {len(customers)} ügyfél, {len(orders)} rendelés")
        return customers, orders

    @task(log_prints=True)
    def compute_ltv(customers: pd.DataFrame, orders: pd.DataFrame) -> pd.DataFrame:
        """Ügyfélenkénti lifetime value számítás."""
        ltv = orders.groupby('customer_id')['amount'].sum().reset_index()
        ltv.columns = ['customer_id', 'ltv']
        result = customers.merge(ltv, on='customer_id', how='left').fillna(0)
        print(result.to_string(index=False))
        return result

    # Főflow: meghívja a subflow-t, majd tovább dolgozza fel az eredményt
    @flow(name="main-analytics-flow", log_prints=True)
    def main_flow():
        # A subflow meghívása ugyanolyan, mint egy normál task hívása
        customers, orders = ingestion_subflow()
        ltv = compute_ltv(customers, orders)
        print(f"\nLTV összesítő: {len(ltv)} ügyfél, összLTV={ltv['ltv'].sum():,.0f} Ft")

    main_flow()
    ```

    !!! note "Subflow és hibakezelés"
        Ha a subflow meghiúsul, a főflow is meghiúsul – pontosan úgy, mint egy normál Python kivétel esetén. Ez lehetővé teszi, hogy a pipeline komplex feltételes logikával épüljön fel, miközben minden lépés naplózott és nyomon követhető a Prefect UI-ban.

### on_failure_hook – hibakezelés és értesítés

31. A Prefect `on_failure_hooks` paramétere segítségével automatikusan futtathatunk kódot, ha egy flow meghiúsul – pl. Slack értesítést küldhetünk:

    ```python
    from prefect import flow, task
    from prefect.states import Failed
    import traceback

    def notify_on_failure(flow, flow_run, state):
        """
        Ez a függvény automatikusan fut, ha a flow meghiúsul.
        Valós rendszerben: Slack webhook, email, PagerDuty API hívás.
        """
        error_msg = str(state.result(raise_on_failure=False))
        print(f"[ALERT] Flow '{flow.name}' meghiúsult!")
        print(f"  Flow run ID: {flow_run.id}")
        print(f"  Hiba: {error_msg[:200]}")
        # Valós példa:
        # requests.post(SLACK_WEBHOOK, json={"text": f":x: {flow.name} failed: {error_msg}"})

    @task
    def risky_task(x: int) -> int:
        if x == 0:
            raise ZeroDivisionError("Nullával való osztás a pipeline-ban!")
        return 100 // x

    # on_failure_hooks: lista of callable – mindegyik meghívódik hiba esetén
    @flow(name="risky-flow", on_failure=[notify_on_failure], log_prints=True)
    def risky_flow(divisor: int = 0):
        result = risky_task(divisor)
        print(f"Eredmény: {result}")

    # Normál futás
    print("=== Normál futás (divisor=5) ===")
    risky_flow(divisor=5)

    # Hibás futás – a hook automatikusan meghívódik
    print("\n=== Hibás futás (divisor=0) ===")
    try:
        risky_flow(divisor=0)
    except Exception as e:
        print(f"A flow meghiúsult (várt): {type(e).__name__}")
    ```

    !!! note "on_failure_hooks vs on_failure_callback (Airflow)"
        Az Airflow-ban a `default_args`-ban megadható `on_failure_callback` hasonló célt szolgál. A Prefect megközelítése modernebb: a hook egy normál Python függvény, amely a flow, flow_run és state objektumokat kapja – könnyen integrálható Slack, PagerDuty, vagy egyedi alerting rendszerekkel.
