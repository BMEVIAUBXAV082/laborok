# 1. feladat – PostgreSQL OLTP forrás és csillag séma ETL

Az adattárházak (Data Warehouse) kiindulópontja mindig egy OLTP (Online Transaction Processing) forrásrendszer, amely normalizált, tranzakcióra optimalizált sémában tárolja az adatokat. Az analitikai lekérdezések számára azonban a normalizált séma hátrányos: sok JOIN-t igényel, és sok kis táblát kell bejárni. Ezért az adatokat egy **csillag sémába** (star schema) transzformáljuk: egy ténytáblából (fact table) és néhány dimenzió táblából (dimension tables) áll, amelyek sugarasan csatlakoznak – innen a neve.

!!! note "Csillag séma alapfogalmak"
    * **Grain** – a ténytábla egy sorának pontos jelentése. Mindig deklarálni kell! Példa: „egy sor = egy rendelési tétel, egy adott napon, egy adott ügyfélnek, egy adott termékből."
    * **Surrogate key (szurrogált kulcs)** – a DWH által generált technikai azonosító (pl. `customer_key = 1, 2, 3...`). Független a forrásrendszertől – akkor is stabil marad, ha az OLTP ID-k változnak.
    * **Natural key (természetes kulcs)** – a forrásrendszer eredeti azonosítója (pl. `customer_id = 42`). Megőrizzük referenciaként, de JOIN-okhoz a surrogate key-t használjuk.
    * **Fact tábla** – mérőszámokat (mennyiség, ár, összeg) és FK hivatkozásokat tartalmaz a dimenziókra. Sok sor, kevés oszlop.
    * **Dim tábla** – leíró attribútumok (név, város, kategória). Ritkán változik, ezért külön tároljuk.

### PostgreSQL OLTP séma feltöltése

12. A második cellában hozzuk létre az OLTP táblákat és töltsük fel szintetikus adatokkal:

    ```python
    import psycopg2                         # PostgreSQL Python kliens könyvtár
    import pandas as pd
    import duckdb
    import random
    from datetime import date, timedelta
    from faker import Faker

    # Kapcsolódás a PostgreSQL konténerhez
    # A 'postgres' hostnév a Docker hálózaton belül a labor04-postgres konténert jelenti
    # A saját gépünkről (kívülről) localhost:5432-n érnénk el, de konténerből konténerbe a service névvel
    conn = psycopg2.connect(
        host="postgres", port=5432,
        user="labor", password="labor",
        dbname="oltp_source"
    )
    conn.autocommit = True   # Minden SQL utasítás automatikusan COMMIT-olódik
    cur = conn.cursor()

    # Korábbi futtatások maradványainak eltávolítása – idempotens újrafuttatáshoz
    # CASCADE: az order_items táblát is törli, amely customers és products-ra hivatkozik
    cur.execute("DROP TABLE IF EXISTS order_items CASCADE")
    cur.execute("DROP TABLE IF EXISTS orders CASCADE")
    cur.execute("DROP TABLE IF EXISTS products CASCADE")
    cur.execute("DROP TABLE IF EXISTS customers CASCADE")

    # OLTP séma: 3NF (harmadik normálalak) – minden adat pontosan egy helyen tárolódik
    cur.execute("""
        CREATE TABLE customers (
            id         SERIAL PRIMARY KEY,    -- AUTO INCREMENT azonosító
            name       TEXT NOT NULL,
            email      TEXT,
            city       TEXT,
            country    TEXT DEFAULT 'Hungary',
            created_at DATE DEFAULT CURRENT_DATE
        )
    """)

    cur.execute("""
        CREATE TABLE products (
            id         SERIAL PRIMARY KEY,
            name       TEXT NOT NULL,
            category   TEXT,
            unit_price NUMERIC(10, 2) NOT NULL,  -- 10 jegyű, ebből 2 tizedes
            created_at DATE DEFAULT CURRENT_DATE
        )
    """)

    cur.execute("""
        CREATE TABLE orders (
            id          SERIAL PRIMARY KEY,
            customer_id INTEGER REFERENCES customers(id),   -- FK a customers táblára
            order_date  DATE NOT NULL,
            status      TEXT DEFAULT 'completed'
        )
    """)

    cur.execute("""
        CREATE TABLE order_items (
            id         SERIAL PRIMARY KEY,
            order_id   INTEGER REFERENCES orders(id),       -- FK az orders táblára
            product_id INTEGER REFERENCES products(id),     -- FK a products táblára
            quantity   INTEGER NOT NULL,
            unit_price NUMERIC(10, 2) NOT NULL              -- Tranzakció idejebeli ár (nem a jelenlegi!)
        )
    """)

    print("OLTP séma létrehozva: customers, products, orders, order_items")

    # Szintetikus adatgenerálás Faker-rel
    fake = Faker('hu_HU')    # Magyar neveket és városokat generál
    random.seed(42)          # Reprodukálható eredmények – ugyanaz az adat minden futásnál

    cities = ['Budapest', 'Debrecen', 'Miskolc', 'Pécs', 'Győr', 'Szeged', 'Nyíregyháza']

    # 200 ügyfél generálása
    customer_rows = [
        (fake.name(), fake.email(), random.choice(cities), 'Hungary',
         date(2022, 1, 1) + timedelta(days=random.randint(0, 730)))
        for _ in range(200)
    ]
    cur.executemany(
        "INSERT INTO customers (name, email, city, country, created_at) VALUES (%s,%s,%s,%s,%s)",
        customer_rows
    )

    # 25 termék, 5 kategóriában
    product_catalog = {
        'Elektronika': ['Laptop', 'Monitor', 'Billentyűzet', 'Egér', 'Headset'],
        'Bútor':       ['Íróasztal', 'Forgószék', 'Könyvespolc', 'Szekrény', 'Kanapé'],
        'Irodaszer':   ['Tűzőgép', 'Iratrendező', 'Toll készlet', 'Jelölőcetli', 'Ollók'],
        'Élelmiszer':  ['Kávé 500g', 'Tea mix', 'Ásványvíz 6×', 'Energia szelet', 'Müzliszelet'],
        'Sport':       ['Jóga szőnyeg', 'Kézisúlyzó', 'Ugrálókötél', 'Rezisztencia szalag', 'Tornazsák'],
    }
    product_rows = []
    for cat, names in product_catalog.items():
        for pname in names:
            price = round(random.uniform(500, 80000), 2)
            product_rows.append((pname, cat, price, date(2022, 1, 1)))
    cur.executemany(
        "INSERT INTO products (name, category, unit_price, created_at) VALUES (%s,%s,%s,%s)",
        product_rows
    )

    # 500 rendelés és ~1500 rendelési tétel
    cur.execute("SELECT id FROM customers")
    cust_ids = [r[0] for r in cur.fetchall()]          # Összes ügyfél ID listája

    cur.execute("SELECT id, unit_price FROM products")
    prod_data = cur.fetchall()                          # [(product_id, unit_price), ...]

    for _ in range(500):
        cust_id = random.choice(cust_ids)
        odate   = date(2023, 1, 1) + timedelta(days=random.randint(0, 364))
        # Szándékosan vegyes státusz – az ETL csak a 'completed' rendeléseket veszi át
        status  = random.choice(['completed', 'completed', 'completed', 'pending', 'cancelled'])
        cur.execute(
            "INSERT INTO orders (customer_id, order_date, status) VALUES (%s,%s,%s) RETURNING id",
            (cust_id, odate, status)
        )
        order_id = cur.fetchone()[0]   # A RETURNING id visszaadja az újonnan létrejött sort

        # 1–4 tétel rendelésenként
        for prod_id, base_price in random.sample(prod_data, random.randint(1, 4)):
            qty   = random.randint(1, 10)
            price = round(float(base_price) * random.uniform(0.9, 1.1), 2)  # ±10% áringadozás
            cur.execute(
                "INSERT INTO order_items (order_id, product_id, quantity, unit_price) VALUES (%s,%s,%s,%s)",
                (order_id, prod_id, qty, price)
            )

    # Összesítés
    print("Szintetikus adatok betöltve:")
    for table in ['customers', 'products', 'orders', 'order_items']:
        cur.execute(f"SELECT COUNT(*) FROM {table}")
        print(f"  {table}: {cur.fetchone()[0]:,} sor")
    ```

    !!! note "Miért SERIAL és REFERENCES?"
        A `SERIAL` típus automatikusan növekvő egész számot generál – ez az OLTP természetes kulcsa. A `REFERENCES` idegen kulcs (FK) kényszert deklarál: az `order_items.product_id` csak olyan értéket vehet fel, amely az `products.id` oszlopban létezik. Ez az adatintegritás alapja az OLTP rendszerben. A DWH-ban ezeket a kényszerekat általában **nem** kényszerítik ki – a sebesség fontosabb.

### DuckDB csillag séma feltöltése

13. A harmadik cellában hozzuk létre a DuckDB dimenzió táblákat:

    ```python
    # In-process DuckDB adatbázis – nincs szükség különálló szerverre
    # A fájl perzisztens: a notebookok között is megmarad az adatbázis
    dcon = duckdb.connect("duckdb")

    # Meglévő táblák törlése újrafuttathatóság érdekében
    for tbl in ['fact_order_item', 'dim_date', 'dim_customer', 'dim_product']:
        dcon.execute(f"DROP TABLE IF EXISTS {tbl}")

    # dim_customer – az ügyfél dimenzió
    # Kulcs különbség az OLTP-hez képest: customer_key (surrogate) mellé megtartjuk a customer_id-t (natural)
    dcon.execute("""
        CREATE TABLE dim_customer (
            customer_key  INTEGER PRIMARY KEY,   -- DWH belső azonosító (1, 2, 3, ...)
            customer_id   INTEGER NOT NULL,       -- Forrásrendszer ID – referencia marad
            name          TEXT,
            city          TEXT,
            country       TEXT
        )
    """)

    # dim_product – a termék dimenzió
    dcon.execute("""
        CREATE TABLE dim_product (
            product_key   INTEGER PRIMARY KEY,
            product_id    INTEGER NOT NULL,
            product_name  TEXT,
            category      TEXT,
            unit_price    DOUBLE   -- Listaár a dimenzióban (nem a tranzakciós ár!)
        )
    """)

    # fact_order_item – a ténytábla (grain: 1 rendelési tétel)
    # Csak FK-k és mérőszámok – leírások a dimenziókban vannak
    dcon.execute("""
        CREATE TABLE fact_order_item (
            order_item_key INTEGER PRIMARY KEY,
            customer_key   INTEGER REFERENCES dim_customer(customer_key),
            product_key    INTEGER REFERENCES dim_product(product_key),
            order_date     DATE,                 -- Dátum direktben (kis DWH esetén dim_date nem szükséges)
            quantity       INTEGER,
            unit_price     DOUBLE,               -- Tranzakció idejebeli ár
            total_amount   DOUBLE                -- Levezetett mező: quantity × unit_price
        )
    """)

    print("DuckDB csillag séma táblák létrehozva.")
    ```

14. A negyedik cellában töltsük fel a dimenzió táblákat PostgreSQL adatokból:

    ```python
    from sqlalchemy import create_engine

    # SQLAlchemy connection string: dialect+driver://user:password@host:port/dbname
    # Ezt a pd.read_sql() igényli – psycopg2 közvetlen connection-t is elfogad, de deprecated
    engine = create_engine("postgresql://labor:labor@postgres:5432/oltp_source")

    # --- dim_customer feltöltése ---
    df_cust = pd.read_sql("SELECT id, name, city, country FROM customers ORDER BY id", engine)
    # Surrogate key generálása: sorrend szerinti egész szám, 1-től indul
    df_cust.insert(0, 'customer_key', range(1, len(df_cust) + 1))
    df_cust.columns = ['customer_key', 'customer_id', 'name', 'city', 'country']
    # DuckDB közvetlenül tud pandas DataFrame-ből INSERT-elni – nincs szükség row-by-row iterálásra
    dcon.execute("INSERT INTO dim_customer SELECT * FROM df_cust")
    print(f"dim_customer: {len(df_cust)} sor betöltve")

    # --- dim_product feltöltése ---
    df_prod = pd.read_sql(
        "SELECT id, name, category, unit_price FROM products ORDER BY id", engine
    )
    df_prod.insert(0, 'product_key', range(1, len(df_prod) + 1))
    df_prod.columns = ['product_key', 'product_id', 'product_name', 'category', 'unit_price']
    dcon.execute("INSERT INTO dim_product SELECT * FROM df_prod")
    print(f"dim_product:  {len(df_prod)} sor betöltve")
    ```

    !!! note "Miért nem az OLTP id-t használjuk surrogate key-ként?"
        Az OLTP rendszer ID-ja változhat: törlés-újraszúrás esetén ugyanaz az objektum más ID-t kaphat, vagy különböző forrásrendszerekből érkező adatoknál ütközhetnek az ID-k. A surrogate key mindig a DWH-on belül generált, garantáltan egyedi, és nem függ a forrásoktól.

### A ténytábla feltöltése

15. Az ötödik cellában valósítsuk meg az ETL transzformációt – a ténytábla feltöltését:

    ```python
    # Staging: rendelési tételek kinyerése PostgreSQL-ből, JOIN-okkal összerakva
    # WHERE status = 'completed': csak teljesített rendelések kerülnek a DWH-ba
    df_orders = pd.read_sql("""
        SELECT
            oi.id              AS order_item_id,
            o.customer_id,
            oi.product_id,
            o.order_date,
            oi.quantity,
            oi.unit_price,
            ROUND(oi.quantity * oi.unit_price, 2) AS total_amount
        FROM order_items oi
        JOIN orders o ON oi.order_id = o.id
        WHERE o.status = 'completed'
          AND oi.quantity > 0       -- Negatív mennyiség kiszűrése (adatminőség)
          AND oi.unit_price > 0     -- Ingyenes tételek kiszűrése
    """, engine)
    print(f"Staging: {len(df_orders):,} tétel (csak completed rendelések)")

    # NK→SK lookup: natural key → surrogate key leképezés
    # set_index('customer_id') → index alapján gyors keresés a map() híváshoz
    sk_cust = dcon.execute(
        "SELECT customer_id, customer_key FROM dim_customer"
    ).df().set_index('customer_id')['customer_key']

    sk_prod = dcon.execute(
        "SELECT product_id, product_key FROM dim_product"
    ).df().set_index('product_id')['product_key']

    # NK→SK feloldás: minden natural key-hez megkeressük a megfelelő surrogate key-t
    # A .map() a Series index alapján végzi a kikeresést – hiányzó kulcsnál NaN-t ad vissza
    df_orders['customer_key'] = df_orders['customer_id'].map(sk_cust)
    df_orders['product_key']  = df_orders['product_id'].map(sk_prod)

    # TODO: Szűrje ki a NULL-okat, és töltse be a fact táblát!
    #
    # Lépések:
    #   1. Szűrjük ki azokat a sorokat, ahol a customer_key VAGY product_key NaN (NULL):
    #      df_fact = df_orders.dropna(subset=['customer_key', 'product_key']).copy()
    #      (Ezek olyan rendelések lennének, amelyekhez nem találtunk dimenziót – hibás adat)
    #
    #   2. Alakítsuk egésszé a FK oszlopokat (map() után float típusú lesz):
    #      df_fact['customer_key'] = df_fact['customer_key'].astype(int)
    #      df_fact['product_key']  = df_fact['product_key'].astype(int)
    #
    #   3. Generáljuk a surrogate key-t:
    #      df_fact.insert(0, 'order_item_key', range(1, len(df_fact) + 1))
    #
    #   4. Töltsük be a DuckDB fact táblába a szükséges oszlopokkal:
    ```

    ??? tip "Megoldási útmutató – kattints a tippekért"
        ```python
        # NULL szűrés: ha egy natural key nem szerepel a dimenzióban, azt a sort kihagyjuk
        # Ez az ETL "reject" logikája – valós rendszerben ezeket hibanaplóba írjuk
        df_fact = df_orders.dropna(subset=['customer_key', 'product_key']).copy()

        # A .map() float-tá alakítja az int oszlopot (NaN kezelése miatt)
        # Az astype(int) visszaalakítja egésszé az INSERT előtt
        df_fact['customer_key'] = df_fact['customer_key'].astype(int)
        df_fact['product_key']  = df_fact['product_key'].astype(int)

        df_fact.insert(0, 'order_item_key', range(1, len(df_fact) + 1))

        df_insert = df_fact[['order_item_key', 'customer_key', 'product_key',
                              'order_date', 'quantity', 'unit_price', 'total_amount']]
        dcon.execute("INSERT INTO fact_order_item SELECT * FROM df_insert")
        print(f"fact_order_item: {len(df_fact):,} sor betöltve")
        ```

        **Miért .map() és nem JOIN?** A pandas `.map(Series)` hatékonyabb, mint egy SQL JOIN: a lookup Series index-alapú keresést végez O(1)-es komplexitással, szemben a pandas merge O(n log n)-es viselkedésével. Nagy adatmennyiségnél ez érzékelhetően gyorsabb.

16. A hatodik cellában ellenőrizzük az eredményt:

    ```python
    # Táblák sorszámainak összesítése – ez lesz a beadandó képernyőkép
    result = dcon.execute("""
        SELECT 'dim_customer'    AS tabla, COUNT(*) AS sorok FROM dim_customer   UNION ALL
        SELECT 'dim_product',              COUNT(*)          FROM dim_product    UNION ALL
        SELECT 'fact_order_item',          COUNT(*)          FROM fact_order_item
    """).df()
    print(result.to_string(index=False))

    # Üzleti ellenőrző lekérdezés: top 5 kategória bevétel szerint
    # Ez a JOIN pontosan azt mutatja, amit a csillag séma megkönnyít
    top = dcon.execute("""
        SELECT
            dp.category,
            COUNT(*)                       AS tetelek,
            ROUND(SUM(f.total_amount), 0)  AS bevetel
        FROM fact_order_item f
        JOIN dim_product dp ON f.product_key = dp.product_key
        GROUP BY dp.category
        ORDER BY bevetel DESC
    """).df()
    print("\nKategóriák bevétel szerint:")
    print(top.to_string(index=False))
    ```
