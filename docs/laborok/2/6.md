# 6. feladat – Dead Letter Queue

Éles streaming rendszerekben mindig előfordulnak feldolgozhatatlan üzenetek: hiányzó mező, rosszul formázott JSON, ismeretlen esemény-típus, negatív szám egy kötelezően pozitív mezőben, stb. Az iparági bevált gyakorlat szerint ezeket **nem szabad csendben eldobni** – helyette egy külön **Dead Letter Queue (DLQ)** topicba kell irányítani, ahol később megvizsgálhatók, javíthatók és újra feldolgozhatók.

!!! note "DLQ pattern"
    A DLQ minta alapja: `try: feldolgoz(msg)` → `except: dlq_producer.send(msg + hiba_oka)`. A hibás üzenet az eredeti tartalommal **és a hibaokksal együtt** kerül a DLQ topicba. Így a hibás üzenetek nem vesznek el, az alap consumer stream folyamatosan fut, és az ops csapat utólag megvizsgálhatja a problémákat.

    ```python
    from kafka import KafkaProducer, KafkaConsumer
    from kafka.admin import KafkaAdminClient, NewTopic
    import json, time, random
    from datetime import datetime
    import pandas as pd

    BOOTSTRAP = "kafka:9092"

    # A DLQ topicot létrehozzuk, ha még nem létezik
    admin = KafkaAdminClient(bootstrap_servers=BOOTSTRAP)
    if "orders-dlq" not in admin.list_topics():
        # Csak 1 partíció: a DLQ-ban általában kis mennyiségű üzenet van, nincs szükség skálázásra
        admin.create_topics([NewTopic("orders-dlq", num_partitions=1, replication_factor=1)])
    admin.close()

    producer = KafkaProducer(
        bootstrap_servers=BOOTSTRAP,
        value_serializer=lambda v: json.dumps(v).encode("utf-8"),
    )

    # 15 üzenet generálása (300–314 order_id tartomány)
    messages = []
    for i in range(300, 315):
        msg = {
            "order_id": i,
            "customer_id": random.randint(1, 100),
            "product_id": random.randint(1, 50),
            "quantity": random.randint(1, 5),
            "unit_price": round(random.uniform(500, 50000), 2),
            "event_time": datetime.utcnow().isoformat(),
        }
        messages.append(msg)

    # Szándékos hibák bevitele – 3 üzenet lesz érvénytelen
    messages[2]["quantity"] = -5       # Negatív mennyiség – üzleti szabály sérti
    messages[5]["unit_price"] = None   # None ár – nem szám, feldolgozáskor hibát okozna
    del messages[9]["product_id"]      # Hiányzó kötelező mező

    for msg in messages:
        producer.send("orders", value=msg)   # Minden üzenetet elküldünk – a hibásakat is!
    producer.flush()
    producer.close()
    print(f"{len(messages)} üzenet elküldve (köztük 3 szándékosan hibás).")
    ```

    ??? tip "Megoldási útmutató – kattints a tippekért"
        ```python
        def process_message(msg: dict) -> bool:
            required_fields = ["order_id", "customer_id", "product_id", "quantity", "unit_price", "event_time"]

            # Szabály 1: kötelező mezők
            for field in required_fields:
                if field not in msg:
                    dlq_producer.send("orders-dlq", value={
                        "original": msg,
                        "error": f"Hiányzó mező: {field}",
                        "ts": datetime.utcnow().isoformat(),
                    })
                    print(f"[DLQ] Hiányzó mező '{field}': order_id={msg.get('order_id')}")
                    return False

            # Szabály 2: quantity pozitív
            if msg["quantity"] <= 0:
                dlq_producer.send("orders-dlq", value={
                    "original": msg,
                    "error": f"Érvénytelen quantity: {msg['quantity']} (kell: > 0)",
                    "ts": datetime.utcnow().isoformat(),
                })
                print(f"[DLQ] Negatív quantity: order_id={msg['order_id']}")
                return False

            # Szabály 3: unit_price érvényes
            if msg["unit_price"] is None or msg["unit_price"] <= 0:
                dlq_producer.send("orders-dlq", value={
                    "original": msg,
                    "error": f"Érvénytelen unit_price: {msg['unit_price']}",
                    "ts": datetime.utcnow().isoformat(),
                })
                print(f"[DLQ] Érvénytelen unit_price: order_id={msg['order_id']}")
                return False

            print(f"[OK] order_id={msg['order_id']}")
            return True
        ```

### Consumer loop DLQ-val

40. Írjuk meg a consumer loop-ot, amely minden üzenetet átad a `process_message`-nek:

    ```python
    # TODO: Hozzunk létre egy KafkaConsumer-t az "orders" topicra:
    #   - group_id: "dlq-consumer"     → egyedi group_id, hogy saját offset-je legyen
    #   - auto_offset_reset: "earliest" → az elejéről olvassa (beleértve a régi üzeneteket)
    #   - consumer_timeout_ms: 4000
    #   - value_deserializer: JSON decode
    #
    # Inicializáljuk a számlálókat:
    #   ok_count = 0
    #   dlq_count = 0
    #
    # A for ciklus minden üzeneténél:
    #   - Ha msg.value dict típusú:
    #     - Hívjuk a process_message(msg.value)-t
    #     - Ha True (sikeres): ok_count += 1
    #     - Ha False (DLQ-ba ment): dlq_count += 1
    #
    # Végén:
    #   - dlq_producer.flush() + dlq_producer.close()  → megvárjuk a DLQ üzenetek kézbesítését
    #   - consumer.close()
    ok_count = 0
    dlq_count = 0
    ```

    ??? tip "Megoldási útmutató – kattints a tippekért"
        ```python
        consumer = KafkaConsumer(
            "orders",
            bootstrap_servers=BOOTSTRAP,
            group_id="dlq-consumer",
            auto_offset_reset="earliest",
            enable_auto_commit=True,
            value_deserializer=lambda v: json.loads(v.decode("utf-8")),
            consumer_timeout_ms=4000,
        )

        ok_count = 0
        dlq_count = 0

        for msg in consumer:
            if isinstance(msg.value, dict):
                if process_message(msg.value):
                    ok_count += 1
                else:
                    dlq_count += 1

        dlq_producer.flush()
        dlq_producer.close()
        consumer.close()
        ```
