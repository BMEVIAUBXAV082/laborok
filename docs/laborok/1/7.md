# 7. feladat: Particionált Parquet és predicate pushdown

A nagy adathalmazok hatékony kezelésének kulcsa a particionálás. Ha az adatokat rendezett könyvtárstruktúrába írjuk (pl. `year=2024/month=03/`), az analitikai motorok képesek a nem releváns partíciókat teljesen kihagyni. Ebben a feladatban particionált Parquet fájlokat írunk DuckDB-vel, és megmérjük a predicate pushdown hatását.

### Notebook létrehozása

1. Hozzunk létre egy új notebookot `feladat7_partitioned.ipynb` néven.

2. Az első cellában inicializáljuk a kapcsolatot és építjük újra a cleaned táblát:

    ```python
    !pip install -q duckdb pyarrow pandas

    import duckdb
    import pandas as pd
    import os
    import time

    duck = duckdb.connect()
    RAW  = "/home/jovyan/data/extract/full"
    PART = "/home/jovyan/data/partitioned"
    os.makedirs(PART, exist_ok=True)

    for t in ["customers", "products", "orders", "order_items"]:
        duck.execute(f"CREATE OR REPLACE TABLE stg_{t} AS SELECT * FROM read_parquet('{RAW}/{t}.parquet')")

    duck.execute("""
        CREATE OR REPLACE TABLE cleaned_orders AS
        SELECT o.*, c.name AS customer_name, c.segment AS customer_segment,
               YEAR(order_date)  AS year,
               MONTH(order_date) AS month
        FROM stg_orders o
        JOIN stg_customers c ON o.customer_id = c.customer_id
        WHERE o.status != 'cancelled'
    """)
    cnt = duck.execute("SELECT COUNT(*) FROM cleaned_orders").fetchone()[0]
    print(f"cleaned_orders: {cnt:,} sor, year/month oszlopokkal bővítve.")
    ```

### Particionált export

3. Exportáljuk az adatokat `year=/month=` könyvtárstruktúrába:

    ```python
    duck.execute(f"""
        COPY cleaned_orders
        TO '{PART}'
        (FORMAT PARQUET, PARTITION_BY (year, month), OVERWRITE_OR_IGNORE TRUE)
    """)
    print(f"Particionált export kész: {PART}")
    ```

### Fájlszerkezet vizsgálata

4. Vizsgáljuk meg, milyen könyvtárstruktúra jött létre:

    ```python
    all_files = []
    for root, dirs, files in os.walk(PART):
        for f in files:
            if f.endswith(".parquet"):
                full_path = os.path.join(root, f)
                size_kb = os.path.getsize(full_path) / 1024
                rel_path = os.path.relpath(full_path, PART)
                all_files.append({"fájl": rel_path, "méret (KB)": round(size_kb, 1)})

    files_df = pd.DataFrame(all_files).sort_values("fájl")
    print(f"Összesen {len(files_df)} partíciófájl\n")
    print(files_df.to_string(index=False))
    print(f"\nÖsszes méret: {files_df['méret (KB)'].sum():.1f} KB")
    ```

### Full scan vs partition-pruned scan

5. Mérjük meg a teljes scan és a szűrt scan különbségét:

    ```python
    print("=== Partition Pruning benchmark ===\n")

    # Full scan – összes partíció
    start = time.time()
    full_cnt = duck.execute(
        f"SELECT COUNT(*) FROM read_parquet('{PART}/**/*.parquet')"
    ).fetchone()[0]
    full_ms = (time.time() - start) * 1000

    # Partition-pruned scan – csak 2024-es adatok
    start = time.time()
    pruned_cnt = duck.execute(
        f"SELECT COUNT(*) FROM read_parquet('{PART}/**/*.parquet') WHERE year = 2024"
    ).fetchone()[0]
    pruned_ms = (time.time() - start) * 1000

    print(f"  Full scan:           {full_cnt:>8,} sor   {full_ms:>8.1f} ms")
    print(f"  Pruned scan (2024):  {pruned_cnt:>8,} sor   {pruned_ms:>8.1f} ms")
    if pruned_ms > 0:
        print(f"  Gyorsulás:           {full_ms / pruned_ms:.1f}×")

    print(f"\n  Pruned / Full aránya: {pruned_cnt / full_cnt * 100:.1f}% az adatoknak olvasva")
    ```

    !!! tip "Predicate pushdown és partition pruning"
        Amikor a DuckDB `WHERE year = 2024` szűrőt lát, a könyvtárstruktúra alapján automatikusan **csak a `year=2024/` alkönyvtárakat** nyitja meg. A többi partíciót teljesen kihagyja, még fájllistázás sem történik rájuk. Ez a **partition pruning** alapja — az olvasott adatmennyiség a szűrő szelektivitásával arányosan csökken.

### Row group statisztikák vizsgálata

6. Vizsgáljuk meg az egyik Parquet fájl belső row group statisztikáit:

    ```python
    import pyarrow.parquet as pq

    # Első partíciófájl megkeresése
    sample_file = None
    for root, dirs, files in os.walk(PART):
        for f in files:
            if f.endswith(".parquet"):
                sample_file = os.path.join(root, f)
                break
        if sample_file:
            break

    if sample_file:
        meta = pq.read_metadata(sample_file)
        schema = pq.read_schema(sample_file)
        print(f"Fájl: {os.path.relpath(sample_file, PART)}\n")
        print(f"Row group-ok száma: {meta.num_row_groups}")
        print(f"Sorok száma:        {meta.num_rows}\n")

        rg = meta.row_group(0)
        print(f"{'Oszlop':<25} {'Null':>6} {'Min':>20} {'Max':>20}")
        print("-" * 75)
        for i in range(rg.num_columns):
            col = rg.column(i)
            stats = col.statistics
            if stats:
                print(f"  {col.path_in_schema:<23} {stats.null_count:>6} "
                      f"{str(stats.min)[:18]:>20} {str(stats.max)[:18]:>20}")
    ```

    !!! note "Column statistics"
        Minden Parquet fájlban minden row group tartalmaz oszlopszintű min/max és null_count statisztikákat. Az analitikai motorok ezeket olvassák el a tényleges adatok megnyitása előtt — ha a `WHERE` feltétel kívül esik a [min, max] intervallumon, a teljes row group kihagyható.
